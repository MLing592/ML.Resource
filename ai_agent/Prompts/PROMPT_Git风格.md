# Git提交信息提示词

## 用途

用于约束解决方案中使用Git提交时的规范，确保后续新增/更改/重构模块时，Git规则保持一致。

## 提交要求
- 如果用户有要求，那就使用其他语言或者翻译为其他语言.
- `commit` 信息使用中文，包括：
  - `plan`：表示计划文档相关，例如 `XX实施计划.md`
  - `docs`：表示说明文档相关，例如 `ReadMe.md`
  - `test`：表示测试相关，例如 `MSTest` / `XUnit`
  - `feat`：表示代码新功能
  - `fix`：表示代码问题修复
  - `chore`：表示一些无关紧要的小改动
  - `style`：表示项目规范的变更，例如 `.editconfig`
  - `comment`：表示专门的一次代码注释改动
  - `refactor`：表示代码出现重构
  - `perf`：表示代码优化
  - `revert`：表示提交回滚
  - `merge`：表示代码合并
  - `build`：表示构建相关，例如 `Directory.Build.props`
  - `ci`：表示 `CI/CD` 流程相关
  - `github`：表示 `GitHub` 相关流程
